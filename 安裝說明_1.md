# LINE 群組「被 Tag 訊息」每日備份 Bot — 安裝說明

## 這個 Bot 會做什麼

1. Bot 待在群組裡，整天默默記錄所有「有 @某人」的訊息
2. 每天晚上 23:55（台灣時間），自動把「你今天被 tag 的訊息」做成**聊天截圖樣式的卡片**
3. 私訊傳到你和 Bot 的一對一聊天室，當作備份

## 先知道的三個限制（LINE 官方規定，任何 Bot 都一樣）

| 你原本想的 | 實際做法 |
|---|---|
| 事後分析一整天對話 | Bot 必須**當天就在群組裡**即時記錄，無法讀取加入前的歷史訊息 |
| 截圖 | Bot 無法操作手機截圖，改用 Flex 卡片畫出「對話泡泡」，效果等同截圖 |
| 用 LINE 的分享功能 | Bot 無法幫人按分享鍵，改由 Bot **直接私訊**給被 tag 的人（對方必須先加 Bot 好友）|

LINE 本身沒有內建這種功能（「Keep 筆記」只能手動轉存），所以一定要用 Bot。

---

## 安裝步驟（約 15 分鐘）

### 第 1 步：申請 LINE Bot（免費）

1. 到 https://developers.line.biz/ 用你的 LINE 帳號登入
2. 建立 Provider → 建立 Channel → 選「**Messaging API**」
3. 進入 Channel 後記下兩個東西：
   - **Channel secret**（在 Basic settings 分頁）
   - **Channel access token**（在 Messaging API 分頁，按 Issue 產生）
4. 在 Messaging API 分頁：
   - 「Allow bot to join group chats」→ 開啟（Enabled）
   - 「Auto-reply messages」→ 關閉

### 第 2 步：部署程式（推薦用免費的 Render.com）

1. 把這個資料夾上傳到你的 GitHub（新建一個 repository）
2. 到 https://render.com 註冊 → New → **Web Service** → 連結你的 GitHub repo
3. 設定：
   - Build Command：`npm install`
   - Start Command：`npm start`
4. 在 Environment 加入兩個環境變數：
   - `LINE_CHANNEL_ACCESS_TOKEN` = 第 1 步拿到的 token
   - `LINE_CHANNEL_SECRET` = 第 1 步拿到的 secret
5. 部署完成後會得到一個網址，例如 `https://xxxx.onrender.com`

### 第 3 步：把網址接回 LINE

1. 回到 LINE Developers → Messaging API 分頁
2. Webhook URL 填：`https://xxxx.onrender.com/webhook`
3. 按 Verify（顯示 Success 即成功）
4. 「Use webhook」→ 開啟

### 第 4 步：開始使用

1. **把 Bot 邀請進群組**（用 Messaging API 分頁的 QR code 加 Bot，再邀進群）
2. **每個想收備份的人，都要掃 QR code 加 Bot 為好友**（沒加好友收不到私訊）
3. 完成！之後群組裡只要有人被 @，當晚 23:55 就會自動收到備份

### 測試

在群組輸入 `!備份`，Bot 會立刻寄出目前累積的備份，不用等到晚上。

## 常見問題

- **有人收不到備份？** → 他還沒加 Bot 好友。
- **想改寄送時間？** → 加環境變數 `DAILY_CRON`，格式如 `0 22 * * *`（每天 22:00）。
- **@All 會被記錄嗎？** → 不會，只記錄 @特定人。
- **免費 Render 會休眠？** → 免費方案閒置會睡著，可用 https://uptimerobot.com 每 10 分鐘 ping 一次你的網址保持清醒（也免費）。

## 本機測試（進階，可跳過）

```bash
npm install
export LINE_CHANNEL_ACCESS_TOKEN=你的token
export LINE_CHANNEL_SECRET=你的secret
npm start
# 另開視窗用 ngrok 建立對外網址
npx ngrok http 3000
# 把 ngrok 給的 https 網址 + /webhook 填到 LINE Webhook URL
```
